CSS Grid

The website is built on basic HTML and CSS, but it uses the new Grid specification of CSS, which allows for two dimensional layouts, so you can define X and Y coordinates for elements.


Defining the grid took quite a while at first, but after defining the sizes, it's much, much easier and faster to create webpages


I used a two letter coordinate system for grid elements, the first letter indicating the row and the second letter the column.


.cAA{ grid-area: aa;}
.cAB{ grid-area: ab;}
.cAC{ grid-area: ac;}
.cAD{ grid-area: ad;}

and so on...

Lab 1.1 Definitions


  • a) HTML Hypertext Markup Language is a markup language used to define webpages and their content.
  • b) CSS: Cascading Style Sheet is a way of defining a style and look of a webpage. It's used to overwrite the user agent stylesheet with your own.
  • c) JavaScript: Command language used to add dynamic content to web pages. Also there are JS based APIs such as jQuery that are used to mimiq server side functionality on the client device and to add pre-made ready to use functionality to JS.
  • d) HTTP: Hypertext Transfer Protocol is a data transfer protocol used in the internet. It uses hyperlinks between nodes to transfer hypertext data.
  • e) HMTL5: Markup language for displaying and structuring content in the world wide web. The HTML5 specification was published in 2014 by the World Wide Web Consortium, W3C.
  • f) Dynamic HTML: Is a term used for a collection of technologies used to create interactive websites. These technologies include things such as HTML, JS, CSS and DOM.

Lab 1.2 Coding conventions


  • Naming: name the files, functions, classes etc. in the same manner consistently, so you or the next author of the code will have an easier time reading and understanding your code.
  • Commenting: Comment your code in a meaningful manner that it provides "value" and needed information. For example if you have a bug or a workaround in the code and you don't comment on it, couple months down the line when you come back to it and spend the same amount of time trying to resolve it and coming down the same conclusion.
  • Formatting & style: Format your code in a sensible matter. If you're the original author of the code, you can define the way of formatting and the rest should follow it. If you work in a company, follow the company formatting policy.
